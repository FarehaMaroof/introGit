GIT AND GIT HUB

GitHub allows developers to create, store, manage, and share their code. Git is version control system and GitHub is a cloud-based platform where you can store, share, and work together with others to write code.

Steps to download and install Git

  1. Download Git from Download git
  2. Select the Type of Operating System and Download
  3. Restart or Reboot the PC
  4. To check if its installed go to VS code and open terminal from top left options.Type :git --versionon terminal and it will show the latest terminal installed

To use git in VS

  1. In Terminal type command :git init.It will initialize in specific folder and keep track of everything in the project
  2. Now 'U' will appear after all the files in that specific folder representing untracked file i.e Git is not able to track their location.yoc can check the same by typing 'git status' it will give list of files that are untracked
  3. To track file write command:git add filename
    and to track all files type:git add
    and the commit by typing :git commit -m 'your msg'
  4. If you are Committing for the first time then you will get following error: *** Please tell me who you are.
    Run git config --global user.email "you@example.com"
    git config --global user.name "Your Name"
    to set your account's default identity.
    Omit --global to set the identity only in this repository.
    So you have to run these commands:
    git config --global user.email "you@example.com"
    git config --global user.name "Your Name"
    and configure it and then again add and commit.
    commit is used to create a log of the changes made so far and keeps track of the changes made so far.
  5. To see number of times committed type commands :git log

Suppose you made some mistake in last commit and you want to delete it and go back to last second commit then

  1. Type git log command and see the log
    select the code of the commit you want to go back to
    eg:commit athe31678ba550gh109eabfd0da4588f3c6
    now copy the code 'athe31678ba550gh109eabfd0da4588f3c6'
    • type command and paste the code : git reset --hard athe31678ba550gh109eabfd0da4588f3c6
    • hard :when you want to completely remove last version
    • type command and paste the code :git reset --soft athe31678ba550gh109eabfd0da4588f3c6
    • soft :When you want to go to required version bt doesnt want to remove the version in which mistake was made.

Git Hub

  1. Create an account on
  2. Click on new button on top left corner
  3. Give name to your repository scroll down and create repository
  4. You will get list of commands to be used

Create a new repository on the command line

  • echo "# VCS" >> README.md
  • git init
  • git add README.md
  • git commit -m "first commit"
  • git branch -M main
  • git remote add origin https://github.com/ABC/VCS.git
  • git push -u origin main

Push an existing repository from the command line:

  • git remote add origin https://github.com/ABC/VCS.git
  • git branch -M main
  • git push -u origin main
  • Now whenever you create a file you just have to type :git add .,the type :git commit -m 'msg' , then type :git push and file will be added to the same repository

If you want to download files of someones's from git hub to your desktop

  • Go to repository,click on code and from drop down select link in HTTPS
  • Open Cmd in your terminal and type :git clone link(which you have copied)
  • now every time new files added or any changes made you just have to type :git pull and data will be downloaded into your files

    Branching:
    Its is used when you want to make some changes in file and test it bt you dont want to manipulate contents of the previous content written on that file

    • After Typing the code ,Write command:git branch
    • Then write command:Git branch Test(It will create two file one to modify and test and one original file which will remain unaltered)
    • Now whatever code you want to extend in original write that code run and decide whether you want it in original or not
    • To switch from one branch to another type :git checkout test(it will get out of test and move to main file)
    • If you want to check which branch you are in type:git branch
      for eg main
      * test       (this shows you are in test file)
    • Once decided if you want to keep the changes in original you can merge by command :git merge test(type the command in main branch)
